BGP (Border Gateway Protocol) Overview
Introduction

    BGP is the protocol used to exchange routing information between different autonomous systems (AS) on the internet.
    It is classified as a path-vector protocol.
    BGP is defined in RFC 4271.

Key Features

    Supports policy-based routing.
    Uses TCP (port 179) for reliable communication.
    Supports route aggregation and filtering.
    Provides loop prevention.
    Used for Internet routing and large enterprise networks.

BGP Types

    External BGP (eBGP): Between different ASes.
    Internal BGP (iBGP): Within the same AS.

BGP Attributes

    AS_PATH: List of ASes the route has traversed.
    NEXT_HOP: Next hop IP address.
    LOCAL_PREF: Local preference for route selection.
    MED (Multi-Exit Discriminator): Influences inbound traffic.
    COMMUNITY: Tag for route grouping and policy application.

BGP Route Advertisement

    BGP routers advertise best paths based on attributes and policies.
    Routes are exchanged via UPDATE messages.

BGP Configuration Basics

    Establish BGP neighbors.
    Define BGP networks.
    Apply route policies and filters.

Example Configuration (Cisco IOS)

plaintext

router bgp 65001
 neighbor 192.168.1.2 remote-as 65002
 network 10.0.0.0 mask 255.255.255.0

BGP Best Path Selection

    Highest LOCAL_PREF.
    Shortest AS_PATH.
    Lowest origin type.
    Lowest MED.
    eBGP routes preferred over iBGP.

Troubleshooting Tips

    Verify BGP neighbor status (show ip bgp summary).
    Check BGP routes (show ip bgp).
    Confirm policies and filters.
    Ensure proper AS and neighbor configurations.
